# Nexus AI 新闻系统 — 后端设计文档

> 最后更新：2026-06-13

---

## 1. 项目概述

基于 FastAPI + SQLAlchemy（异步）构建的 AI 资讯聚合后端，自动采集 Hacker News、OpenAI Blog、Google AI Blog、MIT Technology Review 四个数据源，入库时使用 Claude Haiku 翻译英文内容为中文，提供新闻浏览、搜索、收藏、历史记录及用户管理功能。

---

## 2. 技术栈

| 类别 | 技术 |
|------|------|
| 框架 | FastAPI |
| 数据库 | MySQL 8.0 + SQLAlchemy 2.0（异步） |
| 驱动 | aiomysql |
| 缓存 | Redis |
| 爬虫 | httpx、feedparser |
| 翻译 | Claude Haiku（`claude-haiku-4-5-20251001`），通过 httpx 直接调用代理 |
| 密码加密 | bcrypt |
| 定时任务 | asyncio（内置，随 FastAPI lifespan 启动） |

## 4. 数据库设计

### news 表（核心）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 自增主键 |
| title | VARCHAR(255) | 原文标题 |
| title_zh | VARCHAR(500) | 中文标题（翻译） |
| description | VARCHAR(500) | 原文摘要 |
| description_zh | TEXT | 中文摘要（翻译） |
| content | TEXT | 原文正文 |
| content_zh | TEXT | 中文正文（翻译） |
| image | VARCHAR(255) | 封面图 URL |
| author | VARCHAR(50) | 作者 |
| category_id | INT FK | 分类 ID |
| source_platform | VARCHAR(50) | 来源平台（hackernews/openai/google_ai/mit） |
| source_url | VARCHAR(500) | 原文链接 |
| content_hash | VARCHAR(32) | MD5 去重哈希 |
| views | INT | 浏览量 |
| publish_time | DATETIME | 发布时间 |

### 其他表

- **user** — 用户信息（id、username、password_hash、nickname、avatar 等）
- **user_token** — 认证 Token（支持登出失效）
- **news_category** — 分类表（id、name）
- **favorite** — 收藏（user_id、news_id）
- **history** — 浏览历史（user_id、news_id、created_at）

## 5. 自动采集机制

`main.py` 通过 FastAPI `lifespan` 在服务启动时创建后台 asyncio 任务，**无需 Celery，无需 Redis broker**：

```python
@asynccontextmanager
async def lifespan(app):
    task = asyncio.create_task(_scheduler_loop())  # 每 2 小时采集一次
    yield
    task.cancel()
```

同时提供 `POST /api/news/refresh` 接口，前端可手动触发采集（`BackgroundTasks` 异步执行，接口立即返回）。

### 采集流程

1. `rss_fetcher.py` 拉取三个 RSS 源，`hn_fetcher.py` 调用 HN API
2. `base.save_news()` 用 MD5(title) 去重，新条目调用 `translator.py` 并发翻译三个字段
3. 翻译失败不影响入库，`_zh` 字段留空

### 翻译注意事项

代理 `ANTHROPIC_BASE_URL` 已含 `/v1`，必须用 `httpx` 直接拼 `/messages`，不能用 anthropic SDK（SDK 会重复追加 `/v1` 导致 404）。

## 6. API 接口

### 新闻模块 `/api/news`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/categories` | 数据源列表（全部/HN/OpenAI/Google AI/MIT） |
| GET | `/list` | 新闻列表，支持 `source`/`page`/`pageSize` 过滤 |
| GET | `/detail?id=` | 新闻详情，含中文字段 + 相关推荐 + sourceUrl |
| GET | `/search?keyword=` | 标题/摘要关键词搜索 |
| POST | `/refresh` | 触发后台采集（立即返回） |

详情接口返回字段：`id / title / titleZh / description / descriptionZh / content / contentZh / image / author / source / sourceUrl / publishTime / views / relatedNews`

### 用户模块 `/api/user`

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/register` | 否 | 注册 |
| POST | `/login` | 否 | 登录，返回 token |
| POST | `/logout` | 是 | 登出，Token 立即失效 |
| GET | `/info` | 是 | 获取当前用户信息 |
| PUT | `/update` | 是 | 更新昵称/简介/性别/手机 |
| PUT | `/password` | 是 | 修改密码 |
| POST | `/avatar` | 是 | 上传头像 |

### 收藏 `/api/favorite` / 历史 `/api/history`

标准 CRUD，含列表分页、单条删除、清空全部，均需 Token 认证。

---

## 7. 统一响应格式

所有接口返回：

```json
{ "code": 200, "message": "success", "data": { ... } }
```

`code` 为业务码（前端读此字段），HTTP status_code 与业务码保持一致（如失败同时返回 HTTP 4xx 和 `code: 4xx`）。

---

## 8. 认证机制

- Token 存于 `user_token` 表，有效期 7 天
- 请求头：`Authorization: <token>`
- 登出后 Token 在表中标记失效，立即无法使用
- 登录/注册接口有限流保护（每 IP 每分钟 10 次）

---

## 9. 迁移管理

迁移文件放在 `migrations/versions/`，命名格式 `000N_描述.py`，每个文件实现 `upgrade(conn)` / `downgrade(conn)` 异步函数。

```bash
python migrations/migrate.py            # 执行所有未应用迁移
python migrations/migrate.py --status   # 查看状态
python migrations/migrate.py --rollback # 回滚最后一次
```

**注意：** MySQL 不支持 `ADD COLUMN IF NOT EXISTS`，需用 `information_schema` 检查列是否存在再 ALTER。
